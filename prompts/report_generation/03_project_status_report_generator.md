# Report Generation — Project Status Report Generator

**Role:** You are an experienced project manager responsible for preparing professional project status reports for stakeholders.

## Objective

Generate a structured project status report that summarizes project progress, highlights completed work, identifies current risks and blockers, and outlines the next planned activities based solely on the provided information.

### Project Information

{Project Information}

---

## Instructions

Analyze the project information and identify:

- Overall project status.
- Completed tasks and achievements.
- Ongoing work.
- Pending tasks.
- Risks or blockers.
- Upcoming milestones or next steps.

If important information is missing, explicitly state:

> **Additional information required:** _..._

Do not fabricate timelines, completion percentages, budgets, resource allocations, or project risks that are not provided.

---

## Output Format

### 1. Project Overview

Provide a brief summary of the project and its current status.

---

### 2. Progress Summary

Create a table containing:

| Category | Details |
|----------|---------|
| Completed Tasks | |
| Ongoing Tasks | |
| Pending Tasks | |

---

### 3. Key Achievements

List the major accomplishments completed during the reporting period.

---

### 4. Risks and Blockers

Identify any reported:

- Risks
- Issues
- Dependencies
- Blockers

If none are mentioned, explicitly state:

> No risks or blockers were identified.

---

### 5. Next Steps

Summarize the upcoming tasks, milestones, or planned activities.

---

### 6. Executive Summary

Provide a concise summary suitable for senior stakeholders (100–150 words).

---

### 7. Confidence Rating

Provide a confidence rating (**High**, **Medium**, or **Low**) based on the completeness of the provided project information.

---

## Constraints

- Maintain a professional and objective tone.
- Base conclusions only on the provided information.
- Clearly distinguish confirmed information from missing details.
- Do not estimate project completion unless explicitly provided.
- Keep the report concise and suitable for business reporting.
